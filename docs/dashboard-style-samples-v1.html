Dashboard vs. page style — what to weigh

  • Dashboard wins: everything on one screen, filters update all panels at once, feels like a professional decision tool — great for the desktop-first direction and for data-heavy users (Tharindu persona).
  • Page style wins: simpler for first-time and low-literacy users (Sunil persona) — one question per screen, bigger text, less to scan. The mobile adaptation is also more natural from pages.
  • Middle path (my recommendation): dashboard shell on desktop with these panels, but each panel stays big-typed and single-message; on mobile the same panels stack into the signed-off page flow. No PRD conflict — content, honesty rules, and components are identical, only the desktop composition changes.
