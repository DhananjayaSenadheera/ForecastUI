3 · Data-representation layer (the viz vocabulary, applied honestly)

Where each representation is used, and the rules that keep every one of them honest and readable.

RepresentationWhereAnswersHonesty / a11y rule
Confidence band (range + bold centre)Result W3, W3b · mobile D/D2"What price, how sure?"Marked centre always; never a bare interval; icon+label+reason beside it.
Timeline line chart (history + forecast band)Result W3 · Timeline mobile E"Where's it heading?"Shaded band not a single line; harvest marker; table alternative mandatory.
Seasonality heatmap/bar stripResult W3 · crop detail"Which months pay best?"Intensity scale color-blind-safe (FE-1); needs API #2 history.
Comparison table (sortable)Best crops W4 · Prices W5"Which crop/market wins?"Confidence column shown, never dropped; ranges kept alongside central price.
Grouped bars / small multiplesBest crops W4 · Market compare W5 R2"Compare at a glance"Whiskers carry P10–P90; no dual axes; direct labels.
Factor breakdown (direction table)Result W3 · mobile F"Why this price?"Plain phrases + direction icon+label; no raw feature names/weights.
Data table (text alternative)Every chart"Exact numbers"WCAG: every graphic has an equivalent table; numbers are the product.
Data-viz principles (carried into FE-1):
  1. aOne message per chart — each graphic answers a single question; no kitchen-sink dashboards.
  2. bDirect labelling over legends where space allows (label the line/bar itself).
  3. cTable alternative for every graphic (WCAG 2.2, and farmers are numerate).
  4. dColor-blind-safe encoding planned for FE-1; encoding never relies on hue alone — shape/label/position too.
  5. eNo dual axes, no 3D, no decoration — maximise data-ink ratio; grayscale here proves the layout reads without color.
  6. fUncertainty never dropped — bands, whiskers and confidence labels travel with every number, including inside comparisons.
  7. gProvenance under every chart (HARTI / DOA) — trust is a data-viz requirement here.

5 · Cross-cutting design notes (every frame, both layouts)

  1. AAudio-help everywhere: speaker (🔊, ≥44px) in every header; key blocks (confidence, factors, errors) get their own "Hear this".
  2. BLanguage switcher in every header (සි / த / EN), first-launch forces the choice once.
  3. CTouch/click targets ≥44px; primary buttons 48px; comfortable on desktop pointer and mobile touch alike.
  4. DOne primary action per screen; secondaries are ghost. Max nav depth 2 (header nav desktop / bottom tabs mobile).
  5. ENumbers-first: price is the largest element; confidence never hidden; ranges always show a marked centre.
  6. FProvenance line (HARTI / DOA) under every price and chart.
  7. GFour async states (loading / success / empty / error) for every data view.
  8. HHonest uncertainty is a hard rule: Low/Medium/High → distinct pictograph + label + plain reason; fallbacks labelled "rough estimate", never polished into precision; contract strings frozen, only display labels translated.
i18n / typography tolerance: buttons, nav links, table cells and cards wrap instead of truncating (Sinhala/Tamil run 20–40% longer than English); no fixed-height text box uses overflow:hidden; body ≥16px, price numerals 28–40px; line-height ≥1.7 (Tamil 1.8); no uppercase. Reason/Explanation need reason codes from the API (gap #5) so the client owns translation. Numbers/dates locale-aware (LKR "Rs.", local date format). Desktop's extra width is used for content, not larger fonts — the trilingual type scale is identical across breakpoints.
